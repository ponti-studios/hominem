# CLAUDE.md — packages/db

PostgreSQL access layer. **Server-only** — client code must use `@hominem/rpc` instead.

## Generated types

`src/types/database.ts` is generated by kysely-codegen — never edit it by hand. After any schema change:

```bash
just db-migrate    # runs Goose migrations + regenerates types
```

## Writing migrations

Files live in `migrations/`. Use Goose SQL format — every migration needs both markers:

```sql
-- +goose Up
-- your changes here

-- +goose Down
-- your rollback here
```

Schemas: `auth`, `app`, `ops`. Extensions already installed: `pgcrypto`, `pg_trgm`, `unaccent`, `vector`, `earthdistance`, `ltree`, `fuzzystrmatch`. Do not add `CREATE EXTENSION` for these.

Migrations must be idempotent — CI runs `goose up` twice and checks for pending migrations.

## Multi-step writes

Use `runInTransaction` from `src/transaction.ts` for any operation that touches multiple tables:

```ts
import { runInTransaction } from '@hominem/db';
await runInTransaction(async (trx) => { ... });
```
