## Why

The AI Elements implementation plan is still living as a standalone document even though it describes future product work. It should be moved into OpenSpec so the repo has a single way to plan that effort.

## What Changes

- Capture the AI Elements UI adoption effort as an OpenSpec change.
- Replace the legacy custom plan with an OpenSpec proposal, design, tasks, and capability spec.
- Preserve the phased component adoption, chat integration, and verification work.

## Capabilities

### New Capabilities

- `ai-elements-chat-ui`: Defines the AI Elements-based chat UI capability for the product surfaces that use chat.

### Modified Capabilities

None.

## Impact

- Affected code: chat UI components, shared UI package exports, AI interaction flows, and integration tests.
- Affected systems: chat UX consistency, shared UI primitives, and AI feature presentation.
